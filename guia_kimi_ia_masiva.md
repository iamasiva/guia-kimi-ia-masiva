# Guia IA MASIVA: funciones de Kimi AI

Kimi no es solamente un chatbot para hacer preguntas.

La idea grande es esta: Kimi esta intentando convertirse en un espacio de trabajo completo con IA. No solo te responde, tambien puede ayudarte a crear presentaciones, sitios web, documentos, hojas de calculo, investigaciones profundas, codigo, agentes y hasta habilidades personalizadas para repetir procesos.

En esta guia vamos a ver, una por una, las funciones que aparecen en el panel de Kimi y para que le sirven a una persona que quiere trabajar mas rapido, crear mas contenido, investigar mejor o automatizar tareas.

> Nota para la comunidad IA MASIVA: piensa en esta guia como un mapa rapido. La pregunta no es "que boton tiene Kimi", sino "que trabajo puedo dejar de hacer manualmente gracias a este boton".

## 1. Slides: presentaciones en minutos

![Slides](<Copia de Captura de pantalla 2026-05-23 a la(s) 8.50.41 a.m. 8.png>)

**Para que sirve**

Slides sirve para crear presentaciones profesionales a partir de una idea, un texto largo, un documento o una instruccion simple.

En vez de empezar desde una diapositiva en blanco, puedes darle a Kimi el tema, el publico y el objetivo. La herramienta organiza la informacion, propone una estructura y la convierte en una presentacion visual.

**Que problema resuelve**

La mayoria de personas pierden demasiado tiempo convirtiendo ideas en slides. No es solo escribir: hay que ordenar, resumir, disenar, elegir colores, decidir que va en cada diapositiva y hacer que todo se vea profesional.

Slides reduce ese trabajo manual.

**Ejemplo real**

Imagina que tienes un informe de 30.000 palabras sobre el mercado de vehiculos electricos y necesitas presentarlo a inversionistas. En vez de leer, resumir y disenar todo desde cero, le pides a Kimi que lo convierta en una presentacion de 20 diapositivas con puntos clave, estructura ejecutiva y visuales claros.

**Prompt sugerido**

```text
Convierte este informe en una presentacion ejecutiva de 20 diapositivas para inversionistas. Quiero que tenga una narrativa clara, datos clave, riesgos, oportunidades y una conclusion accionable.
```

## 2. Websites: crear sitios web hablando con la IA

![Websites](<Copia de Captura de pantalla 2026-05-23 a la(s) 8.50.41 a.m..png>)

**Para que sirve**

Websites permite generar sitios web completos usando lenguaje natural, referencias visuales, imagenes de diseno o instrucciones sobre lo que quieres construir.

La promesa es simple: describes lo que quieres y Kimi te ayuda a convertir esa idea en una pagina funcional.

**Que problema resuelve**

Crear una web normalmente implica diseno, maquetacion, codigo, responsive, estructura, estilos, componentes y a veces backend. Para alguien sin conocimientos tecnicos, eso puede ser una barrera enorme.

Con Websites, la IA ayuda a acelerar el prototipo y a pasar de idea a version funcional mucho mas rapido.

**Ejemplo real**

Un emprendedor tiene un mockup de una landing para su producto. Lo sube a Kimi, explica que necesita una pagina responsive con formulario, autenticacion y conexion a base de datos. Kimi genera una primera version funcional que puede probar, editar y publicar.

**Prompt sugerido**

```text
Crea una landing page responsive para este producto. Usa el estilo del mockup, agrega seccion de beneficios, precios, testimonios, formulario de registro y un CTA claro.
```

## 3. Docs: documentos largos, mejor estructurados

![Docs](<Copia de Captura de pantalla 2026-05-23 a la(s) 8.50.41 a.m. 2.png>)

**Para que sirve**

Docs sirve para crear, revisar, editar, resumir y convertir documentos. Puede trabajar con formatos como Word, PDF, Markdown y contenidos tecnicos con ecuaciones o estructura compleja.

**Que problema resuelve**

Mucho trabajo profesional vive atrapado en documentos largos: contratos, informes, manuales, propuestas, investigaciones, politicas internas y documentos tecnicos.

El problema no es solo leerlos. El problema es encontrar lo importante, comparar versiones, detectar inconsistencias y convertirlos en entregables utiles.

**Ejemplo real**

Un abogado debe revisar cinco contratos de 100 paginas. Kimi puede ayudar a identificar clausulas clave, riesgos, diferencias entre documentos y generar un resumen comparativo con observaciones.

**Prompt sugerido**

```text
Revisa estos contratos y crea una tabla con clausulas importantes, riesgos, diferencias entre versiones y recomendaciones concretas.
```

## 4. Deep Research: investigacion profunda sin perderse en mil pestanas

![Deep Research](<Copia de Captura de pantalla 2026-05-23 a la(s) 8.50.41 a.m. 3.png>)

**Para que sirve**

Deep Research esta pensado para investigaciones exhaustivas. No se queda en una respuesta corta: busca, compara, sintetiza y convierte informacion compleja en un reporte estructurado.

**Que problema resuelve**

Investigar bien toma tiempo. Hay que revisar fuentes, cruzar datos, separar ruido de informacion util y armar una conclusion con criterio.

Deep Research ayuda a pasar de busquedas sueltas a un analisis mas ordenado.

**Ejemplo real**

Un gestor de portafolio quiere analizar una empresa tecnologica antes de su IPO. Kimi puede recopilar noticias, contexto financiero, competidores, riesgos, tendencias y preparar un reporte con analisis FODA, graficos y conclusiones.

**Prompt sugerido**

```text
Haz una investigacion profunda sobre esta empresa. Analiza mercado, competidores, riesgos, modelo de negocio, proyecciones y oportunidades de inversion. Entrega el resultado como reporte ejecutivo.
```

## 5. Sheets: hojas de calculo con formulas y modelos

![Sheets](<Copia de Captura de pantalla 2026-05-23 a la(s) 8.50.41 a.m. 4.png>)

**Para que sirve**

Sheets ayuda a crear hojas de calculo, modelos financieros, tablas, formulas, graficos y estructuras de analisis en Excel o CSV.

**Que problema resuelve**

Muchas personas saben que necesitan una hoja de calculo, pero no saben como construirla bien. Las formulas avanzadas, modelos financieros, analisis de sensibilidad y dashboards pueden tomar horas o dias.

Kimi puede acelerar esa construccion.

**Ejemplo real**

Un CFO necesita un modelo DCF para evaluar una adquisicion. Describe los supuestos principales, y Kimi genera una hoja con formulas conectadas, escenarios, graficos y variables editables.

**Prompt sugerido**

```text
Crea un modelo financiero DCF para evaluar una adquisicion. Incluye supuestos, flujo de caja proyectado, WACC, valor terminal, sensibilidad y graficos.
```

## 6. Agent Swarm: muchos agentes trabajando en paralelo

![Agent Swarm](<Copia de Captura de pantalla 2026-05-23 a la(s) 8.50.41 a.m. 5.png>)

**Para que sirve**

Agent Swarm permite dividir tareas grandes en subtareas y asignarlas a multiples agentes especializados que trabajan en paralelo.

**Que problema resuelve**

Hay trabajos que son demasiado grandes para resolverlos de forma lineal: analizar muchas empresas, revisar cientos de documentos, construir un reporte extenso, investigar mercados o comparar competidores.

Un enjambre de agentes permite repartir el trabajo: unos investigan, otros analizan, otros redactan, otros verifican y otro agente sintetiza.

**Ejemplo real**

Una consultora necesita analizar 100 fintechs en Latinoamerica. Un grupo de agentes recopila informacion, otro compara datos, otro redacta perfiles y otro arma conclusiones estrategicas.

**Prompt sugerido**

```text
Divide esta investigacion en subtareas. Necesito perfiles de empresas, comparacion competitiva, tendencias del mercado, riesgos, oportunidades y un reporte ejecutivo final.
```

## 7. Kimi Code: IA para trabajar con repositorios y terminal

![Kimi Code](<Copia de Captura de pantalla 2026-05-23 a la(s) 8.50.41 a.m. 6.png>)

**Para que sirve**

Kimi Code es la parte orientada a desarrollo. Sirve para analizar repositorios, entender codigo, ejecutar comandos, refactorizar, corregir errores y automatizar tareas tecnicas.

**Que problema resuelve**

En proyectos grandes, entender el codigo toma mucho tiempo. Migrar patrones, corregir bugs, escribir pruebas o refactorizar puede ser repetitivo y riesgoso.

Kimi Code actua como un asistente que puede leer el contexto del proyecto y trabajar con la terminal o el IDE.

**Ejemplo real**

Un desarrollador hereda un repositorio grande y necesita migrar llamadas API de callbacks a async/await. Kimi Code identifica archivos, aplica cambios, ejecuta pruebas y corrige errores de forma iterativa.

**Prompt sugerido**

```text
Analiza este repositorio y migra todas las llamadas API basadas en callbacks a async/await. Ejecuta las pruebas y corrige los errores hasta que todo pase.
```

## 8. Kimi Claw: agente persistente para tareas recurrentes

![Kimi Claw](<Copia de Captura de pantalla 2026-05-23 a la(s) 8.50.41 a.m. 7.png>)

**Para que sirve**

Kimi Claw funciona como un agente persistente en la nube. La idea es que pueda ejecutar tareas programadas, mantener continuidad y usar habilidades para automatizar procesos.

**Que problema resuelve**

Muchas tareas no son de una sola vez. Hay cosas que quieres hacer todos los dias, todas las semanas o cuando cambie cierta informacion: monitorear mercados, revisar noticias, preparar reportes, hacer seguimiento de competidores o generar resumentes recurrentes.

**Ejemplo real**

Un trader quiere recibir cada manana un analisis de Bitcoin. Configura una tarea para revisar precio, noticias, tendencias, patrones historicos y enviar un resumen a las 7:00 a.m.

**Prompt sugerido**

```text
Cada dia a las 7:00 a.m., analiza Bitcoin: precio, tendencia, noticias relevantes, niveles clave y escenarios posibles. Entrega un resumen claro con conclusion accionable.
```

## 9. Presets: instrucciones reutilizables

![Presets](<Copia de Captura de pantalla 2026-05-23 a la(s) 8.59.24 a.m. 4.png>)

**Para que sirve**

Presets permite guardar configuraciones o estilos de trabajo para no tener que repetir las mismas instrucciones cada vez.

**Que problema resuelve**

Si usas IA en serio, te pasa algo: empiezas a repetir el mismo contexto una y otra vez.

"Responde como analista financiero".
"Usa tono ejecutivo".
"Dame tablas".
"Evalua riesgos".
"No seas generico".

Un Preset guarda ese comportamiento para reutilizarlo.

**Ejemplo real**

Un equipo de marketing crea un Preset llamado "Analista SEO". Cada vez que analiza una pagina, Kimi ya sabe que debe revisar palabras clave, competidores, estructura, intencion de busqueda y recomendaciones.

**Prompt sugerido**

```text
Crea un preset llamado Analista SEO. Debe analizar paginas web, detectar oportunidades de posicionamiento, evaluar competidores y entregar recomendaciones en tabla.
```

## 10. Professional Data: contexto profesional para respuestas mas utiles

![Professional Data](<Copia de Captura de pantalla 2026-05-23 a la(s) 8.59.24 a.m..png>)

**Para que sirve**

Professional Data permite guardar informacion sobre tu rol, industria, preferencias, contexto laboral y datos relevantes para que Kimi responda con mayor personalizacion.

**Que problema resuelve**

Una IA sin contexto suele responder de forma generica. Pero no es lo mismo responderle a un estudiante, a un abogado, a un CFO, a un creador de contenido o a un analista financiero colombiano.

Con Professional Data, Kimi puede adaptar sus respuestas a tu perfil.

**Ejemplo real**

Un analista financiero configura que trabaja en inversiones, que su mercado es Colombia y que usa metodologias cuantitativas. Cuando pregunta por TES, CDT o fintech, Kimi puede responder con contexto mucho mas cercano a su realidad profesional.

**Prompt sugerido**

```text
Guarda este contexto profesional: soy analista de inversiones en Colombia, me interesan fintech, TES, CDT, AFP y valoracion por DCF. Usa ese contexto cuando me des analisis financieros.
```

## 11. Skills: habilidades especializadas para tareas concretas

![Skills](<Copia de Captura de pantalla 2026-05-23 a la(s) 8.59.24 a.m. 2.png>)

**Para que sirve**

Skills son habilidades reutilizables que extienden lo que el agente puede hacer. Funcionan como paquetes de conocimiento o workflows especializados.

**Que problema resuelve**

No todo se resuelve con un prompt nuevo. A veces necesitas un proceso: transformar contenido, analizar datos, aplicar una metodologia, conectar herramientas o seguir pasos especificos.

Una Skill encapsula ese proceso.

**Ejemplo real**

Un creador de contenido instala una Skill para reutilizar articulos. Le das un blog post y la Skill genera un hilo de X, un post de LinkedIn, una pieza para Instagram y un bloque para newsletter.

**Prompt sugerido**

```text
Usa una skill de reutilizacion de contenido para convertir este articulo en: hilo de X, post de LinkedIn, caption de Instagram y newsletter corta.
```

## 12. Document to Skills: convertir documentos en habilidades

![Document to Skills](<Copia de Captura de pantalla 2026-05-23 a la(s) 9.02.01 a.m..png>)

**Para que sirve**

Document to Skills permite transformar documentos en habilidades reutilizables. Es decir, convertir conocimiento escrito en un proceso que la IA pueda ejecutar.

**Que problema resuelve**

Muchas empresas tienen su conocimiento en PDFs, manuales, documentos internos, metodologias y guias. El problema es que esos documentos no ejecutan nada: solo estan ahi.

Con esta funcion, ese conocimiento puede convertirse en una Skill que otros miembros del equipo usan despues.

**Ejemplo real**

Una firma consultora tiene una metodologia de valoracion de startups en un documento de 50 paginas. Lo sube a Kimi y lo convierte en una Skill. Ahora cualquier analista puede aplicar esa metodologia de forma consistente.

**Prompt sugerido**

```text
Convierte este documento en una skill reutilizable. La skill debe aplicar la metodologia paso a paso, pedir los datos necesarios y entregar un analisis final estandarizado.
```

# Como entender Kimi en una sola idea

Kimi esta construyendo algo mas grande que un chat.

Puedes verlo como una combinacion de:

- Un creador de presentaciones.
- Un generador de sitios web.
- Un editor y analista de documentos.
- Un investigador profundo.
- Un constructor de hojas de calculo.
- Un agente de codigo.
- Un sistema de agentes paralelos.
- Una biblioteca de habilidades.
- Un asistente con memoria profesional.

La clave no esta en usar todas las funciones al tiempo. La clave esta en identificar que parte de tu trabajo es repetitiva, lenta o pesada, y elegir la funcion correcta para delegarla.

---

# Ruta recomendada para empezar

Si estas empezando con Kimi, yo lo probaria en este orden:

1. **Docs** para resumir o analizar documentos largos.
2. **Slides** para convertir esos documentos en presentaciones.
3. **Deep Research** para investigaciones que requieran multiples fuentes.
4. **Sheets** para modelos, tablas o analisis numerico.
5. **Presets** para guardar tus instrucciones recurrentes.
6. **Skills** para automatizar procesos repetibles.
7. **Agent Swarm** y **Kimi Claw** cuando ya tengas workflows mas avanzados.

# Cierre para IA MASIVA

La mayoria de personas sigue usando la IA como si fuera Google con esteroides: preguntan, copian una respuesta y ya.

Pero herramientas como Kimi muestran hacia donde va esto: la IA como sistema de trabajo.

No solo responde.
Investiga.
Resume.
Disena.
Programa.
Organiza.
Automatiza.
Recuerda contexto.
Y convierte documentos en procesos reutilizables.

Ese es el cambio importante.

El que aprenda a combinar estas funciones no va a tener "un chatbot".
Va a tener una infraestructura personal de productividad con IA.
